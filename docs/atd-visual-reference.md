# ATD Website Visual Reference

Use this guide when creating hero or section visuals for AlphaTrack Digital website pages.

## Canonical Brand Palette

The PDF brand guide is the source of truth for Route 1.

- Deep blue: `#003399`
- Cyan: `#00AFEF`
- Green: `#33CC99`
- White: use for primary text and logo contrast.
- Greyscale dark/light: use for surfaces, borders, muted text, and backgrounds.

Native ATD UI must use only the brand palette above and controlled neutrals. Avoid close substitutes such as brighter blues, softened cyans, or off-brand greens unless they are generated by opacity from the canonical colors.

## Reference Family

The preferred visual language is inspired by modern tracking/data product illustrations like Wetracked-style assets, adapted into original ATD visuals.

Key traits:
- Dark navy or near-black background.
- Central product object: monitoring hub, dashboard panel, shield, signal core, or device-like card.
- Neon data flow: cyan, deep blue, and green signal trails.
- Floating source/event nodes connected to the hub.
- Soft grainy glows, blurred motion streaks, and subtle dotted/halftone texture.
- Rounded dark UI cards with faint borders and deep shadows.
- Minimal readable text; use short labels only when useful.

## ATD Adaptation Rules

- Use ATD colors only by default: cyan `#00AFEF`, green `#33CC99`, deep blue `#003399`, plus white and greyscale neutrals.
- Do not introduce magenta, pink, orange, yellow, lime, violet, amber, rose, or other non-brand accent colors unless the user explicitly asks for a specific visual reason.
- Third-party brand colors are allowed only inside third-party tool logos or embedded partner/platform marks. Do not use those colors for ATD-native cards, icons, glows, charts, chips, or backgrounds.
- Prefer tokenized colors such as `atd-blue`, `atd-cyan`, `atd-green`, `primary`, `secondary`, `background`, `card`, `muted`, and `foreground`. Avoid hardcoded near-brand values.
- Gradients should move through the canonical brand colors only, normally blue to cyan to green.
- UI radius should feel precise and technical: small controls/chips `8-10px`, cards `12-16px`, large CTA containers up to `20px`. Avoid oversized pill/card radii unless the brand guide or component purpose requires it.
- Show business meaning, not decoration: leads, trials, demo bookings, CRM updates, pipeline, CAC, LTV:CAC, MRR.
- Avoid copying third-party logos or layouts exactly. Use generic labels such as `Ads`, `CRM`, `GA4`, `API`, `MQL`, `Trial`, `Demo`.
- Prefer code-native SVG/CSS visuals for page heroes so they are reusable, responsive, and editable across expertise pages.
- Use this same code-native visual style inside sections when a visual is needed; avoid stock photos for expertise sections unless the user explicitly requests photography.
- For future expertise pages, keep the same system but swap labels and event cards to fit the vertical.

## SaaS Hero Direction

The SaaS page should use:
- A central dark monitoring hub or dashboard device.
- Source nodes feeding the hub from ads, CRM, analytics, API, and qualified lead signals.
- SaaS metric tiles such as `Trials`, `Pipeline`, `CAC`, and `LTV:CAC`.
- A white or bright halo behind the hub only if it improves contrast against the dark page.
